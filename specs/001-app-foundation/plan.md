**Branch**: `001-app-foundation` | **Date**: 2026-06-29 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/001-app-foundation/spec.md`

## Summary

Build the complete application shell for SpatialMind AI: a four-region dashboard
(Navbar / Sidebar / Map Viewport / StatusBar), an interactive Leaflet map with OSM and
satellite basemaps, collapsible sidebar with responsive auto-collapse, dark/light theme
with persistence, and all map UX states (loading overlay, error overlay + Retry).
The foundation MUST be production-ready: zero TS/ESLint errors, WCAG 2.1 AA
accessible, mobile-responsive from 320 px, and secured with HTTP security headers.

No GIS analysis, routing, AI, or data import/export is in scope.

## Architecture Overview

```
Browser
  └── Next.js App Router (/)
        └── layout.tsx
              ├── ThemeProvider     ← applies `dark` class to <html>
              └── QueryProvider     ← TanStack Query client (idle Phase 1)
                    └── page.tsx
                          └── DashboardLayout
                                ├── Navbar
                                │     ├── Toolbar
                                │     └── ThemeToggle ──→ themeStore (Zustand)
                                ├── Sidebar ──────────→ dashboardStore (Zustand)
                                │     └── SidebarToggle
                                ├── MapContainer (dynamic import, ssr: false)
                                │     └── MapCore (Leaflet)  ──→ mapStore (Zustand)
                                │           ├── TileLayer
                                │           ├── ZoomControl
                                │           ├── ScaleControl
                                │           └── LayerSwitcher
                                │     ├── MapLoadingOverlay
                                │     └── MapErrorOverlay
                                └── StatusBar ───────→ mapStore (read: coords, zoom)
```

### Data Flow

```
User moves cursor → Leaflet mousemove → useCoordinates (throttled 16ms)
  → mapStore.setLastCoords() → StatusBar re-renders (coordinate display)

User clicks ThemeToggle → useTheme.toggle() → themeStore (persisted)
  → ThemeProvider applies/removes 'dark' class on <html>
  → All Tailwind dark: variants activate

User clicks SidebarToggle → useSidebar.toggle() → dashboardStore
  → Sidebar width CSS transition animates (300ms)
  → DashboardLayout grid reflows → MapContainer resizes
  → Leaflet map.invalidateSize() called after transition

Viewport < 768px → useBreakpoint detects → useSidebar.autoCollapse()
  → dashboardStore saves previousDesktopState → sidebar hides
Viewport ≥ 768px → useSidebar.restoreDesktop() → dashboardStore reads previousDesktopState
```

## State Management Strategy

### Zustand Store Slices

**themeStore** (`features/theme/store/themeStore.ts`)
```
State:   theme: 'light' | 'dark'
Actions: setTheme(theme), toggle()
Persist: localStorage key 'spatialMind:theme'
```

**dashboardStore** (`features/dashboard/store/dashboardStore.ts`)
```
State:   sidebarState: 'expanded' | 'collapsed'
         desktopSidebarPreference: 'expanded' | 'collapsed'
         isMobile: boolean
Actions: setSidebarState(state), toggleSidebar()
         autoCollapseForMobile(), restoreDesktopState()
Persist: localStorage key 'spatialMind:sidebar' (desktopSidebarPreference only)
```

**mapStore** (`features/map/store/mapStore.ts`)
```
State:   center: LatLng
         zoom: number
         activeBasemapId: string
         mapStatus: 'idle' | 'loading' | 'ready' | 'error'
         lastKnownCoords: LatLng | null
         errorMessage: string | null
Actions: setCenter(latlng), setZoom(n), setActiveBasemap(id)
         setMapStatus(status), setLastKnownCoords(latlng), setError(msg), retry()
Persist: none (map view resets on reload — Phase 1 assumption)
```

### Map Status FSM

```
idle ──(map init starts)──→ loading
loading ──(tiles ready)───→ ready
loading ──(tile error)────→ error
error ──(retry clicked)───→ loading
```

### React Query

QueryProvider is initialised in `app/providers.tsx` with default stale time of 5 min.
No queries exist in Phase 1. Future features add query hooks inside their feature folder.

## GIS Module Architecture

### Leaflet Initialization Pattern

```
MapContainer (React Server Component safe)
  ↓ next/dynamic({ ssr: false })
MapCore (runs only in browser)
  ↓ useEffect(() => { map = L.map(ref.current) }, [])
  ↓ useMapStatus: sets mapStatus = 'loading' on init
  ↓ map.on('load', () => setMapStatus('ready'))
  ↓ tileLayer.on('tileerror', () => setMapStatus('error'))
```

### Leaflet Map Resize on Sidebar Toggle

When the sidebar collapses/expands, the map's container div changes width.
Leaflet does not auto-detect this. After the 300 ms CSS transition completes,
`map.invalidateSize()` MUST be called. This is handled in `useMapInstance`:

```
useSidebar onChange → setTimeout(300ms) → map.invalidateSize()
```

### Basemap Configuration

```typescript
// features/map/constants/basemaps.ts (shape — no implementation code)
type BasemapConfig = {
  id: string
  name: string
  urlTemplate: string
  attribution: string
  maxZoom: number
}
// Two entries: 'osm-street' | 'esri-satellite'
```

### Future GIS Module Extension Points

- `MapCore` accepts a `children` prop — future features render Leaflet layers as
  children (e.g., `<VectorLayer />`, `<HeatmapLayer />`)
- `mapStore` has reserved action slots: `addLayer()`, `removeLayer()`, `setLayerVisibility()`
  — to be implemented in future milestones
- `features/` parallel structure: `features/analysis/`, `features/routing/`,
  `features/ai/`, `features/import/` — each self-contained

## Risk Analysis

| Risk | Probability | Impact | Mitigation |
|---|---|---|---|
| Leaflet SSR crash (window not defined) | High | High | `dynamic({ ssr: false })` at MapContainer; never import Leaflet outside MapCore |
| Tailwind dark mode flash on load | Medium | Medium | Inline script in `<head>` reads localStorage and sets `dark` class before paint |
| CSS transition conflicts with Leaflet resize | Medium | Medium | Call `map.invalidateSize()` in setTimeout(300) after transition end |
| CSP blocking map tiles | Medium | High | Enumerate tile CDN domains before prod deploy; test CSP in staging |
| Coordinate display performance regression | Low | Medium | 16 ms throttle in `useCoordinates`; profile with React DevTools |
| shadcn/ui CSS variable conflicts in dark mode | Low | Medium | Use only semantic Tailwind tokens; never use raw CSS values in components |
| Playwright E2E flakiness on map load timing | Medium | Low | Use `waitForSelector` on map-ready state; add retry in Playwright config |
| Bundle size bloat from transitive deps | Low | Medium | Run bundle analyzer after M1; set size budget in next.config.ts |

## Performance Strategy

| Goal | Strategy |
|---|---|
| Leaflet out of initial bundle | `dynamic({ ssr: false })` — Leaflet is a separate chunk loaded after hydration |
| No unused CSS | Tailwind v4 tree-shakes by default; no CSS-in-JS runtime |
| Sidebar animation GPU-accelerated | CSS `transition: width 300ms ease` — compositor-only, no layout thrash |
| Coordinate update sub-16ms | Throttle `mousemove` handler; update Zustand store (no React re-render for non-subscribed components) |
| Static shell SSR | `app/page.tsx` can be statically rendered; only map is client-only |
| Font optimization | Use `next/font` with subset loading |
| Image optimization | `next/image` for any raster assets; SVG for logo |
| Bundle budget | Add `@next/bundle-analyzer`; alert if initial JS > 200 KB gzipped |

## Accessibility Strategy

| Requirement | Implementation |
|---|---|
| Keyboard navigation | Logical Tab order: Skip-link → Navbar → Sidebar → Map → StatusBar |
| Map controls ARIA | `aria-label="Zoom in"`, `aria-label="Switch basemap"` etc. on all Leaflet controls |
| Theme toggle | `aria-pressed={theme === 'dark'}` + `aria-label="Toggle dark mode"` |
| Sidebar toggle | `aria-expanded={sidebarState === 'expanded'}` + `aria-label="Toggle sidebar"` |
| Coordinate display | `aria-live="polite"` — announces updates without interrupting focus |
| Mobile overlay | shadcn `Sheet` uses Radix Dialog — includes focus trap, `aria-modal`, Escape to close |
| Color contrast | Tailwind CSS variable tokens designed for AA contrast in both themes |
| Skip link | Visually hidden `<a href="#map">Skip to map</a>` at top of DOM |
| Focus indicators | Tailwind `focus-visible:ring-2` on all interactive elements |
| Motion sensitivity | Sidebar transitions use `prefers-reduced-motion` media query override |

## Responsive Strategy

| Viewport | Sidebar Behavior | Navigation |
|---|---|---|
| < 768 px (mobile) | Hidden; Sheet overlay via hamburger | Hamburger in Navbar → Sheet |
| 768–1023 px (tablet) | Collapsed strip (icon-only) | Sidebar toggle in Sidebar |
| ≥ 1024 px (desktop) | Expanded (default) or collapsed per user preference | Sidebar toggle in Sidebar |

**Breakpoint implementation**:
- Tailwind: `hidden md:flex` for desktop sidebar visibility
- `useBreakpoint(768)`: `window.matchMedia('(max-width: 767px)')` for JS logic
- Sidebar width: desktop expanded = `16rem`, collapsed = `3.5rem` (icon strip)
- DashboardLayout grid: `grid-cols-[auto_1fr]` with sidebar `width` transitioning
- MapContainer: `flex-1 min-h-0` to fill remaining space at all widths
